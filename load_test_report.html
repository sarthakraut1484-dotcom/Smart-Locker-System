Critical Security Vulnerability Audit URGENT

A full-scale static code security audit and architectural penetration analysis was performed on the Firebase data plane, web deployment, and hardware ESP32 firmware integration. Multiple Severe (Tier 1) vulnerabilities were discovered.

1. Global Firebase Data Exposure

Files: database.rules.json & firestore.rules

Both the Realtime Database and Firestore are configured with ".read": true, ".write": true and allow read, write: if true; respectively. This means any unauthenticated bot or malicious user on the public internet can view, modify, steal, or delete the entire database. This must be restricted to authenticated users immediately.

2. Web Client Authentication Bypass

File: login.js

Admin authentication is evaluated on the client-side JavaScript. Hardcoded credentials (Admin@0861) are exposed to anyone inspecting the site's code. Furthermore, the dashboard assigns admin privileges via a local sessionStorage object. An attacker can use browser dev-tools to inject {"isAdmin": true} into their session and completely bypass the login screen without a password.

3. Hardware/IoT Firmware Vulnerabilities

File: SmartLocker_EarlyTermination.ino

  • Hardcoded Secrets: The FIREBASE_SECRET is baked directly into the code in plaintext. Anyone dumping the firmware flashes gains root database control.
  • Insecure Comm TLS: wifiClient.setInsecure() is utilized, willfully disabling standard SSL/TLS certification checks. The ESP32 is unprotected against network interception or spoofing.
  • Hardware Brute-Force Risk: The checkPIN() logic has no exponential backoff timeframe mapping or lock-out mechanism. An attacker can infinitely brute-force 4-digit PINs directly on the touchscreen.
