ASEP 2 SMART LOCKER SYSTEM - COMPLETE UPGRADE CHANGELOG
======================================================
This file documents every major technological upgrade, security patch, and blockchain integration applied to the Smart Locker project.

1. LOAD TESTING & PERFORMANCE AUDITING
--------------------------------------
- Simulated 50 concurrent active users via 'autocannon'.
- 4,500+ requests per second handled with ~10ms latency.
- Lighthouse Score: 91/100 Visual Rendering.
- 0ms Total Blocking Time (TBT).

2. HARDENED FIREBASE SECURITY RULES
-----------------------------------
- Fixed universal read/write access vulnerability.
- Implemented strict rules requiring 'request.auth != null'.
- Mapped sessions to specific UUIDs for data isolation.

3. WEBCRYPTO ADMINISTRATOR GATEWAY (SHA-256)
--------------------------------------------
- Replaced insecure plain-text admin password checks.
- Admin login now uses SHA-256 WebCrypto digests.
- Passwords are never stored or transmitted in plain text.

4. HARDWARE ANTI-BRUTE-FORCE LOCKOUT
-------------------------------------
- ESP32 tracks failed PIN attempts.
- 3 failed attempts trigger a mandatory 30-second localized lockout.
- Prevents robots/attackers from guessing PINs at the locker.

5. HARDWARE-LEVEL IOT BLOCKCHAIN INTEGRATION
--------------------------------------------
- Injected 'mbedtls' SHA-256 cryptography into ESP32 firmware.
- Every locker opening generates a cryptographic block.
- Blocks are linked via previous-hash to form an immutable ledger in Firebase.

6. ZERO-TRUST BLOCKCHAIN VERIFICATION ENGINE
--------------------------------------------
- Added "Verify Blockchain Integrity" button in the admin panel.
- Live-checks the entire database ledger history for tampering.
- Instantly detects if any log has been manually modified (Compromised X).

7. AUTO-DYNAMIC PRICING ALGORITHM
----------------------------------
- Admin-configurable pricing engine (Surge Threshold + Multiplier).
- Spikes prices by 20% (default) during peak occupancy (>90%).
- Real-time Firestore sync propagates prices to users in <500ms.

8. AUTO-FREE LOCKER ON SESSION EXPIRY
--------------------------------------
- Front-end registers automatic reset timers on booking.
- Lockers revert to AVAILABLE status instantly when duration expires.
- No manual admin intervention needed to clear old sessions.

9. ZERO-DELAY REAL-TIME SYNC ARCHITECTURE
-----------------------------------------
- Replaced manual refreshes with Firebase 'onSnapshot' listeners.
- Guarantees data consistency across ESP32, Admin, and User Dashboards.

10. END-TO-END SHA-256 PIN HASHING
-----------------------------------
- PINs are hashed in the browser using WebCrypto BEFORE database storage.
- ESP32 hashes keypad entry and compares the hash (One-Way Security).
- Raw PINs are NEVER stored in the cloud.

11. TIME-SENSITIVE QR CODE EXPIRY
----------------------------------
- QR codes now embed a cryptographic UNIX timestamp.
- Booking page rejects any scan older than 10 minutes.
- Prevents screenshot-based booking fraud.

12. HARDWARE TLS CERTIFICATE PINNING
-------------------------------------
- ESP32 firmware pinned to ISRG Root CA X1 (Google/Firebase Root).
- Neutralizes Man-in-the-Middle (MITM) network attacks completely.

13. ADVANCED HTTP SECURITY HEADERS
-----------------------------------
- Updated vercel.json with CSP, HSTS, and X-Frame-Options.
- Protects the web application against XSS and Clickjacking.

14. BOOKING ENGINE RATE LIMITING
---------------------------------
- Blocks rapid-fire booking attempts from a single user.
- Acts as a defense against bot-spamming.

15. MANDATORY SESSION AUTO-EXPIRY (2HR)
----------------------------------------
- User sessions are strictly limited to a 2-hour window.
- Auto-logout triggers to wipe local storage and tokens.

16. AUTH ERROR SANITIZATION (ANTI-ENUMERATION)
-----------------------------------------------
- Standardized all error messages to generic "Incorrect email or password".
- Prevents attackers from identifying which emails are valid users.

17. IMMUTABLE ADMIN TRANSACTION LOGGING
----------------------------------------
- Every admin action (Open, Reset, Pricing) is written to 'admin_logs'.

18. SECURE IN-PAGE PIN MODAL

- Replaced insecure system alert popups with a custom UI overlay.
